1. Inicio

Descripción General del Proyecto: El sistema de reservas permitirá a los clientes reservar mesas en línea, gestionar la disponibilidad de las mesas y optimizar el servicio en el restaurante.

Objetivo: Desarrollar un sistema eficiente de reservas, administración de mesas y gestión de clientes para mejorar la experiencia en el restaurante.

Alcance: El sistema cubrirá la gestión de reservas, la administración de mesas, la visualización de disponibilidad y la interacción con los clientes.

Tecnologías Utilizadas: Utilizaremos las herramientas de diseño como Lucidchart, Draw.IO para la creación de los diagramas.

2. Marco Teórico

Información del Restaurante:

  • Ubicación y contactos principales.
  • Perfil de clientes (familias, parejas, empresas).

Análisis Estructurado: Explicación del paradigma del análisis estructurado para la gestión de procesos de reservas, con enfoque en el modelo esencial: Modelo Ambiental y Modelo de Comportamiento (reservas, cancelaciones, disponibilidad).

Análisis Orientado a Objetos: Principios básicos como abstracción, encapsulamiento, herencia y polimorfismo, aplicados en la creación de clases como Reservas, Mesas, Empleados, etc. Este paradigma facilita la creación de un sistema modular y escalable.

Herramientas para el diseño y análisis: Uso de software como Draw.io y Lucidchart para la creación de diagramas de flujo, diagramas de clases, etc., que ayudan a visualizar el sistema y sus interacciones.

3. Análisis del Sistema

Descripción de los Requisitos Funcionales:

  • Clientes:
    • Registro y actualización de datos personales.
    • Visualización de disponibilidad de mesas en tiempo real.
    • Realización y cancelación de reservas con confirmación inmediata.
  • Administradores/Empleados:
    • Gestión de la ocupación de mesas.
    • Generación de informes sobre reservas y clientes.
    • Asignación de tareas al personal y actualización del inventario.

Requisitos No Funcionales:

  • Usabilidad: Interfaz intuitiva para cualquier tipo de usuario.
  • Seguridad: Protección de datos sensibles a través de encriptación.
  • Eficiencia: Respuesta ágil en tiempo real, incluso con múltiples usuarios.

Modelo Ambiental: El sistema se conecta con inventarios de cocina, sistema externo de pagos y una base de datos centralizada.

Modelo de Comportamiento: Interacciones entre clientes, empleados y el sistema, además de la automatización en la gestión de mesas y actualizaciones en el sistema.

Además de sus funciones básicas, el sistema también se enfoca en ser fácil de usar y seguro, protegiendo la información de los clientes con métodos avanzados de seguridad.

4. Diseño del Sistema

Componentes principales:

  • Gestión de Reservas: Permite a los clientes ver las mesas disponibles y hacer o cancelar sus reservas.
  • Administración: Diseñado para que los empleados puedan organizar el restaurante, revisar informes y gestionar la ocupación.
  • Interfaz de usuario: Diseño adaptado tanto para computadoras como para teléfonos móviles.

Diagramas:

  • Diagrama de Clases: Incluye entidades como Cliente, Reserva, Mesa, Pedido y Detalle Pedido.
  • Diagrama de Contexto: Describe las interacciones clave entre el cliente, el empleado y los sistemas externos.
  • Diagrama ER: Modela los datos relacionales.
  • Diagrama Relacional: Describe las interacciones clave para una posible base de datos.

Para visualizar el funcionamiento del sistema, usamos herramientas como StarUML para crear diagramas que representan los elementos principales y sus interacciones.

5. Explicacion del proyecto en Video

Duración: Entre 10 y 15 minutos.

  • Explicación de cómo se creó el sistema usando herramientas como StarUML y Visual Paradigm.
  • Demostración del Diagrama de Contexto y/o el Diagrama de Clases.
  • Descripción de las herramientas utilizadas y las funciones clave del sistema.

6. Conclusiones

Resumen de los beneficios del sistema diseñado:

  • Optimiza la gestión de mesas, la asignación de horarios y la interacción entre los clientes y el restaurante.
  • Mejora la experiencia del cliente al permitir la reserva anticipada de mesas.
  • Facilita la administración de la ocupación del restaurante en tiempo real.

Observaciones sobre la implementación de los paradigmas: El análisis estructurado es útil para procesos secuenciales, mientras que el análisis orientado a objetos proporciona mayor flexibilidad y escalabilidad.

Este proyecto combinó paradigmas técnicos y flexibles para diseñar un sistema eficiente y escalable. Futuras integraciones podrían incluir funciones avanzadas como sugerencias automáticas basadas en horarios demandados o herramientas de análisis predictivo.
